# Daisy 1.0.2 — bug fixes, audio reliability, security & UX

## Fixed

- Mic switch mid-recording (AirPods plug/unplug, USB mic, sound output re-route) no longer silently loses audio. If the new device has a different sample rate or channel count, Daisy continues writing into a second file alongside the original instead of dropping every buffer into a hidden error counter.
- Two back-to-back calendar meetings used to silently record into one session with the first meeting's title. Now the first meeting saves and the second starts fresh as its own session.
- Calendar event auto-stop was technically wired up but never actually firing because the meeting binding got cleared during session start. Fixed — auto-stop now arms correctly when bound to a calendar event.
- Test summary in Settings now respects the chosen Summary language — picking "Русский" no longer produces an English preview.
- Summary picker is honoured end-to-end: if you choose a specific language, you get that language regardless of what the transcript was in. "Auto" still detects from content as before.
- Settings → Test summary preview now renders the topical outline (it was hidden before, only the one-line lede was shown).
- Anthropic Sonnet was quietly falling back to the transcript's language for the summary and only honouring the picker for the follow-up. Tightened the prompt to write every field in the chosen language.
- "Next steps" / "Следующие шаги" used to appear twice — once as an outline section and once as the action-items checklist. Now actions live only in the checklist.
- Summary section headers now match the summary language: a Russian summary shows "Встреча / Следующие шаги / Ответ клиенту" instead of English headers above Russian content.

## Changed

- Calendar settings (auto-start, auto-stop, menu-bar label, grace period) moved from Settings → Capture into Connections → Calendar, alongside the Apple/Google account rows. One place for the whole calendar story.
- Auto-stop on calendar event end is on by default for new installs. If you explicitly turned it off, your choice is preserved.
- "Apple Calendar — Connected" row no longer says "Connected" twice; the green badge in the section header already carries that.
- Apple Intelligence summary now includes the same topical outline as cloud providers (3–5 sections with flat bullets). Sub-bullets remain cloud-only.

## New

- Daisy warns within 30 seconds if system audio capture is on but no sound is reaching it. This usually means Bluetooth headphones — macOS can't loop those back into Screen Capture. Use built-in speakers, wired headphones, or BlackHole if you need the remote side recorded.
- One-time heads-up at session start if your default output is Bluetooth, so you can switch before the meeting begins.
- If you change output device mid-session, Daisy rebuilds the system audio capture against the new default and surfaces a toast that it's continuing.
- When a meeting app (Zoom, Meet, Teams, Webex) launches while Daisy is already recording, you see a clear toast instead of silent no-op.
- Empty `system_audio.caf` placeholder is created at session start, so post-session debugging can tell "capture never armed" from "capture received nothing".

## Reliability & security

- Hardened the local MCP server, Google OAuth flow, and Sparkle update path after a senior security review. No user-facing changes; the threat model is documented in SECURITY.md.
- Prompt injection guard: instructions spoken by meeting attendees can no longer hijack the summary or the follow-up draft. The transcript is now treated as untrusted data and Daisy redacts credentials and unknown URLs.
- Audio render thread no longer allocates memory on every buffer — eliminates a class of priority-inversion glitches on long sessions or busy Macs.
- Cloud summary parsing is more tolerant: schema variants from Anthropic / OpenAI that previously failed JSON validation now degrade gracefully instead of losing a couple of minutes of work.
- WhisperKit and FluidAudio are pinned to specific commits, so an upstream force-push can no longer brick recording silently.

## Internals

- Pre-launch source-available release — repository at github.com/d5zn/daisy-app is public-readable now, with README, SECURITY.md, issue templates, and PR template.
