# Daisy 1.0.5.2 — tag picker, less retyping

## Changed

- **"Client" tags renamed to "Tags".** Same single-string-per-session model, but the user-facing name no longer presumes the meeting is with an external client — works for "Q3 planning", "Onboarding", "Personal" or anything else you want to group by.
- **History sidebar: tags as a selector, not a chip row.** A long tag list used to push the session list down the sidebar. Now there's a single "All tags ▾" dropdown above the session list with every tag in use (counts shown alongside) plus an "Untagged" bucket. Picking one filters; picking again clears.
- **Session detail: pick from existing tags.** The editable tag field in the session header now has a chevron-dropdown that lists every tag already used across your sessions, so you can pick "Mediacube" once instead of risking three buckets — "Mediacube", "mediacube", and "Mediacube " — depending on what you typed. Free-text input still works for new tags.

## Migration

- Sessions saved by 1.0.5 with `daisy_client:` in their frontmatter are read transparently and rewritten as `daisy_tag:` on the next tag edit. No manual cleanup needed.
