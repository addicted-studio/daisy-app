# Daisy 1.0.7.19

- Settings → Summary now tells you exactly why a provider is unavailable instead of a bare "Unavailable". For Apple Intelligence it names the actual cause — not turned on, still downloading, this Mac isn't eligible, or (a common one) your Mac and Siri are set to different languages — with what to do about it.
- Privacy, enforced in code: the on-device audio engines (speaker diarization, voice detection, the Parakeet dictation engine) are now hard-blocked from the network except during explicit model downloads. A damaged model cache can no longer trigger a silent re-download mid-meeting — it fails visibly instead.
- The Faster dictation engine (Parakeet) picks up ~8% quicker decoding from the updated on-device audio stack.
- New in About: "Get beta updates" — opt in to receive the newest builds first. Off by default; everyone else keeps getting stable releases only.
