# Daisy 1.0.7.27 — update reliability, follow-up scrolling, and polish

## Fixed

- Automatic update checks now run on a defined hourly schedule. Previously the check cadence was unspecified, so some installs could go a long time without noticing a new build was available.
- The Follow-up section now scrolls all the way through long drafts. It was being clipped at its frame, so the bottom of a longer follow-up couldn't be reached.

## Changed

- The widget's "Hide for 24 hours" is now "Hide for today" — it hides until the next midnight and comes back the following day, instead of a rolling 24-hour window.
- The Home view content is now width-capped and centered to match the Settings pages, instead of stretching edge to edge.
- Tidied the Permissions screen: simpler "Don't remind me" wording, fewer redundant section footers, and the Full Disk Access badge now reads "Optional".
- The dictation history "Clear" button is now styled neutrally rather than as a destructive red action.
- "Send Log Report…" now includes more diagnostics — update-checker state, per-session audio-capture outcomes, and auto-stop decisions — so problems can be pinned down from a single report.
