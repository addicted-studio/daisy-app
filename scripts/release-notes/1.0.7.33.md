# Daisy 1.0.7.33 — safer summaries, fuller Russian

## Fixed

- **No more confident summaries from a failed recording.** If a meeting captured almost no audio — most often because a Bluetooth headset like AirPods took over the mic and system audio — Daisy no longer generates a summary and follow-up from an empty or garbled transcript. Instead it shows a clear warning suggesting the built-in mic and speakers. The raw transcript is still saved, and you can summarize it by hand if you want.
- Selected text is now clearly visible in the light theme (transcript and summary).
- Small caption fixes: the empty-recording note and the activity heatmap legend read correctly in Russian.

## Changed

- **Summary language now matches Apple Intelligence.** When the on-device Apple model is your summary provider, the language picker only offers languages it can actually write. If an unsupported language (e.g. Russian) is already set, Daisy warns that summaries will come out in English and suggests a cloud provider.
- Choosing a different interface language during first-run setup now relaunches Daisy so the whole app switches over immediately.
- Fuller Russian localization across the app.
- The built-in log report now records the current audio route (mic and speaker devices), which helps diagnose capture problems.
