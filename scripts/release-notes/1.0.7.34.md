# Daisy 1.0.7.34 — calendar fixes, faster startup

## Fixed

- **Your Day shows your meetings again.** If you'd given Daisy calendar access but the day view still said "No meetings today or tomorrow," Daisy now checks your live calendar permission on launch instead of a cached flag that could get stuck on "off." Apple Calendar events — including meetings synced in from Google — show up as expected again.

## Changed

- **Faster startup.** Daisy was loading the transcription model twice on every launch — a redundant warm-up pass on top of the real load — which is what left the "Loading model…" notice hanging for a long time, especially right after an update. That extra pass is gone, so the model is ready sooner. (The very first run after install still downloads and compiles the model once; that part is unavoidable.)
