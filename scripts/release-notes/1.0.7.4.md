# Daisy 1.0.7.4 — lighter live transcription, Bluetooth-safe recording, closable window

## Added

- **Lighter live transcription — new "Lite" mode, now the default (build 46).** Settings → Transcription → Live transcription is now a three-way choice: Full / Lite / Off. Lite trims the live decode (fewer Neural Engine workers, greedy sampling, no temperature fallbacks) and runs every ~3.5s instead of every 2s — much lighter on long meetings and on battery, with a slightly rougher live preview. On macOS 26 Lite uses Apple's on-device SpeechAnalyzer, whose model lives in the system asset catalog, so it adds nothing to Daisy's memory; on macOS 14–15, or for languages/locales it doesn't support, Lite transparently falls back to a light Whisper profile. The transcript written on Stop is always full-quality Whisper (with diarization) regardless of this setting, and dictation always uses live. Existing users who had live transcription on are migrated to Lite; those who had it off stay off.
- **Close button on the recording window (build 46).** The menu-bar popover now has an ✕ to dismiss it. Recording keeps running in the background — reopen the window any time from the menu-bar icon.

## Changed

- **Consent reminder moved to the bottom of the recording window (build 46).** The "Always get consent when transcribing others" strip now sits at the bottom instead of pushing the transcript down from the top.

## Fixed

- **Recording no longer goes silent when you connect Bluetooth mid-meeting (build 46).** Previously, connecting AirPods (or any Bluetooth output) while recording on the built-in mic could let macOS quietly switch the input to the Bluetooth microphone, which often captured silence — the timer kept running but your voice wasn't recorded. Daisy now keeps recording on the microphone it started with when a Bluetooth device appears for output, and surfaces a clear warning if an input ever stops delivering audio instead of silently recording nothing. To record through a Bluetooth mic on purpose, pick it in Settings.
